# Jira Gap and Bug Handling

Agents should not leave concrete gaps implicit.

Create or propose:
- `Story` for missing product or platform work
- `Bug` for a defect, regression, or mismatch between expected and actual behavior
- `Spike` for unresolved research or decision work
- `Subtask` only when the work clearly belongs under an existing parent item

Use this when:
- a review finds acceptance criteria not covered by the current story
- implementation exposes missing UX, validation, or rollout work
- documentation and code drift in a way that needs tracked follow-up

Do not create issues for:
- purely speculative ideas with no current execution value
- duplicate work already tracked elsewhere
