# Sprint Review

Review each story against:
- acceptance criteria
- actual merged repo state
- tests and validation evidence
- Jira subtask/status accuracy

Review flow:
1. list target sprint stories
2. inspect merged repo artifacts for each story
3. confirm acceptance criteria are covered
4. reconcile Jira status and subtasks
5. create follow-up Story, Bug, or Spike for real gaps

Output should distinguish:
- implemented and verified
- implemented but Jira not reconciled
- missing behavior
- new follow-up work created
